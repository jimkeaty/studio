## 4. Known Production Edge Cases (Resolved)
We identified and fixed two specific edge cases related to commission tier matching:
1. **Exceeding Top Tier:** If an agent's YTD production exceeded the ceiling of their highest tier, the system previously returned `null`. It now safely falls back to the highest defined tier.
2. **Empty Tiers Array:** If an agent profile had `commissionMode: 'team_default'` but an empty `tiers[]` array in Firestore, the API could fail. We added a final fallback to `STANDARD_TIERS` and a `console.warn` to flag these profiles in Firebase logs.
